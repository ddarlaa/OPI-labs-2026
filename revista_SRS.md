## 1. Введение
**Проект:** Reverso.net  
**Версия документа:** 1.0

Целью данного документа является описание требований к веб-приложению Reverso.net. Система представляет собой набор лингвистических инструментов: переводчик, контекстный словарь, проверка орфографии.

## 2. Общее описание
**Акторы (Роли):**
* **Гость:** Незарегистрированный пользователь.
* **Зарегистрированный пользователь:** Пользователь с аккаунтом.
* **Premium-пользователь:** Платный подписчик.
* **Администратор:** Управление системой.

## 3. Спецификация требований

### 3.1. Требования пользователей (User Requirements - UR)

| ID        | Описание требования                                               | Приоритет | Оценка (ч) |
|:----------|:------------------------------------------------------------------|:----------|:----------:|
| **UR-01** | Пользователь хочет переводить текст с одного языка на другой.     | Must      |     -      |
| **UR-02** | Пользователь хочет видеть примеры использования слов в контексте. | Must      |     -      |
| **UR-03** | Пользователь хочет проверять текст на ошибки.                     | Should    |     -      |

### 3.2. Требования владельцев (Business Rules - BR)

| ID        | Описание требования                                               | Приоритет | Оценка (ч) | Аргументация                       |
|:----------|:------------------------------------------------------------------|:----------|:----------:|:-----------------------------------|
| **BR-01** | Монетизация через баннерную рекламу для бесплатных пользователей. | Must      |     20     | Интеграция AdSense/RTB блоков.     |
| **BR-02** | Конвертация пользователей в Premium подписку.                     | Must      |     40     | Разработка лендинга и ограничений. |

### 3.3. Функциональные требования (Functional Requirements - FR)

| ID        | Описание требования                                                                         | Атрибуты RUP                               | Оценка (ч) | Аргументация                                    |
|:----------|:--------------------------------------------------------------------------------------------|:-------------------------------------------|:----------:|:------------------------------------------------|
| **FR-01** | **Машинный перевод.** Система должна принимать текст, определять язык и возвращать перевод. | **Priority:** Must<br>**Complexity:** High |    160     | UI + API интеграция + обработка ошибок.         |

|:---|:---|:---|:---:|
| **NFR-01** | **Скорость.** Отклик перевода < 1 сек. | Must | 40 |

### Описание диаграммы
Гость (незарегистрированный пользователь) может выполнять перевод текста (включает определение языка), искать контекст и проверять орфографию. Зарегистрированный пользователь наследует права гостя и дополнительно может сохранять слова и покупать подписку.
